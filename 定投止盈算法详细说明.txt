# 定投止盈算法详细说明

## 📋 算法概述
文件路径: `/Users/m672372/Documents/fund_system_ai/src/bussiness/全局智能定投处理/redeem.py`
主要函数: `redeem(user: User, plan_detail: FundPlanDetail) -> bool`

## 🎯 算法核心目标
智能判断是否执行止盈操作，通过多重条件确保在合适时机锁定收益

## 🔄 算法执行流程

### 1. 初始化阶段
```python
# 1.1 日志记录开始
logger.info(f"开始执行止盈算法，用户：{customer_name}")

# 1.2 获取基金基本信息
fund_code = plan_detail.rationPlan.fundCode
fund_info = get_all_fund_info(user, fund_code)
fund_name = fund_info.fund_name

# 1.3 检查基金是否可赎回
if not fund_info.can_redeem:
    logger.info(f"{fund_name}不可赎回")
    return True
```

### 2. 基础信息获取
```python
# 2.1 获取组合账户信息
sub_account_no = plan_detail.rationPlan.subAccountNo
sub_account_name = plan_detail.rationPlan.subAccountName

# 2.2 获取银行份额信息（带异常处理）
try:
    shares = get_bank_shares(user, sub_account_no, fund_code)
except Exception as e:
    logger.warning(f"获取银行份额信息失败: {e}")
    shares = []  # 使用空列表继续处理
```

### 3. 资产详情获取
```python
# 3.1 获取资产详情
asset_detail = get_fund_asset_detail(user, sub_account_no, fund_code)
if asset_detail is None:
    logger.info(f"组合{sub_account_no}的{fund_name}{fund_code}资产为空。Skip ..........")
    return True

# 3.2 提取关键资产指标
plan_assets = asset_detail.asset_value  # 资产价值
fund_type = fund_info.fund_type  # 基金类型
constant_profit_rate = asset_detail.constant_profit_rate  # 当前收益率
on_way_transaction_count = asset_detail.on_way_transaction_count  # 在途交易数

# 3.3 计算投资次数
times = plan_assets // fund_amount  # 投资次数（资产价值/定投金额）
volatility = fund_info.volatility  # 基金波动率
```

### 4. 收益率计算
```python
# 4.1 计算当前收益率
current_profit_rate = constant_profit_rate if constant_profit_rate is not None else 0.0

# 4.2 获取估值增长率
estimated_change = fund_info.estimated_change if fund_info.estimated_change is not None else 0.0

# 4.3 计算预估收益率
estimated_profit_rate = current_profit_rate + estimated_change

# 4.4 获取排名数据
rank_100 = fund_info.rank_100day  # 100日排名

logger.info(f"收益率计算：当前收益率{current_profit_rate}%，估值变化{estimated_change}%，预估收益率{estimated_profit_rate}%")
logger.info(f"其他指标：波动率{volatility}%，100日排名{rank_100}，投资次数{times}")
```

### 5. 基础条件检查
```python
# 5.1 检查份额是否为空
if shares == []:
    return False

# 5.2 收益率过低检查（小于1%不执行止盈）
if estimated_profit_rate < 1.0:
    logger.info(f"组合{sub_account_no}的{fund_name}{fund_code}的收益率{estimated_profit_rate}小于1.0.")
    return True
```

### 6. 🎯 核心止盈逻辑

#### 6.1 趋势门槛检查
```python
# 6.1.1 获取净值数据
est_nav = getattr(fund_info, 'estimated_value', None)  # 估算净值
prev_nav = getattr(fund_info, 'nav', None)  # 前一日净值
nav5 = getattr(fund_info, 'nav_5day_avg', None)  # 5日均值

# 6.1.2 转换为数值
try:
    est_val = float(est_nav) if est_nav is not None else (float(prev_nav) if prev_nav is not None else None)
    nav5_val = float(nav5) if nav5 is not None else None
except Exception:
    est_val = None
    nav5_val = None

# 6.1.3 趋势检查：只有净值高于5日均值时才允许止盈
if est_val is None or nav5_val is None:
    logger.info(f"止盈趋势门槛检查：缺少净值数据，跳过止盈")
    return True
```

#### 6.2 动态止盈点计算
```python
# 6.2.1 计算止盈点：取波动率和3.0%中的较大值
stop_rate = max(float(volatility), 3.0)
logger.info(f"组合{sub_account_no}的{fund_name}{fund_code}波动率={volatility:.2f}%，设置止盈点={stop_rate:.2f}%（不低于3.0%）")

# 6.2.2 QDII基金特殊处理
if asset_detail.fund_type == 'a' and estimated_profit_rate > 3.0:
    logger.info(f"QDII基金{fund_name}{fund_code}预估收益{estimated_profit_rate}%，赎回0费率份额，实际止盈点:3.0%")
    sell_0_fee_shares(user, sub_account_no, fund_code, shares)
    return True

# 6.2.3 基本止盈条件
if estimated_profit_rate > stop_rate:
    logger.info(f"基金{fund_name}{fund_code}预估收益{estimated_profit_rate}%，实际止盈点:{stop_rate}%，执行止盈")
    sell_low_fee_shares(user, sub_account_no, fund_code, shares)
    return True
else:
    logger.info(f"基本止盈条件检查：预估收益{estimated_profit_rate}% <= 止盈点{stop_rate}%，不满足条件")
```

### 7. 💰 银行卡余额相关止盈

#### 7.1 获取银行卡余额
```python
# 7.1.1 尝试从资产API获取余额
try:
    asset_response = GetMyAssetMainPartAsync(user)
    if asset_response.Success and asset_response.Data:
        CurrentRealBalance = asset_response.Data.get('HqbValue', 0.0)
        logger.info(f"从资产API获取HqbValue: {CurrentRealBalance}")
    else:
        raise Exception("资产API调用失败")
except Exception as e:
    logger.warning(f"获取HqbValue失败，回退到原银行卡信息: {str(e)}")
    bank_card_info = user.max_hqb_bank    
    CurrentRealBalance = bank_card_info.CurrentRealBalance
    logger.info(f"银行卡余额：{CurrentRealBalance}")
```

#### 7.2 余额阈值配置
```python
# 7.2.1 从环境变量或配置文件读取阈值
BANK_BALANCE_THRESHOLD = get_bank_balance_threshold()  # 默认300,000.0
PROFIT_THRESHOLD_FOR_LOW_BALANCE = get_profit_threshold_for_low_balance()  # 默认1.0%

logger.info(f"余额阈值配置 - 银行卡余额阈值: {BANK_BALANCE_THRESHOLD}, 低余额止盈阈值: {PROFIT_THRESHOLD_FOR_LOW_BALANCE}%")
```

#### 7.3 低余额止盈条件
```python
# 7.3.1 条件1：普通基金低余额止盈
if (estimated_profit_rate > PROFIT_THRESHOLD_FOR_LOW_BALANCE and 
    CurrentRealBalance < BANK_BALANCE_THRESHOLD and 
    fund_type == '000' and 
    "QDII" not in fund_name and 
    times < 5.0 and 
    estimated_change > 0.5):
    
    logger.info(f"低余额止盈条件触发 - 余额:{CurrentRealBalance},阈值:{BANK_BALANCE_THRESHOLD}")
    logger.info(f"基金{fund_name}{fund_code}预估收益{estimated_profit_rate}%，实际止盈点:{PROFIT_THRESHOLD_FOR_LOW_BALANCE}%")
    logger.info(f"投资次数:{times},估值增长率:{estimated_change}%")
    
    sell_usable_non_zero_fee_shares(user, sub_account_no, fund_code, shares)
    return True
```

#### 7.4 指数基金特殊止盈
```python
# 7.4.1 条件2：指数基金低余额止盈
if (estimated_profit_rate > 3.0 and 
    CurrentRealBalance < BANK_BALANCE_THRESHOLD and 
    fund_type in ['001','002'] and 
    rank_100 is not None and 
    times < 5.0):
    
    logger.info(f"指数基金低余额止盈条件触发 - 余额:{CurrentRealBalance},阈值:{BANK_BALANCE_THRESHOLD}")
    logger.info(f"基金{fund_name}{fund_code}(类型:{fund_type})预估收益{estimated_profit_rate}%，实际止盈点:3.0%")
    logger.info(f"100日排名:{rank_100},投资次数:{times}")
    
    sell_usable_non_zero_fee_shares(user, sub_account_no, fund_code, shares)
    return True
```

### 9. 🎯 止盈条件总结

#### 核心止盈条件：
1. **收益率止盈**：预估收益率 > max(波动率, 3.0%)
2. **QDII基金止盈**：预估收益率 > 3.0% 
3. **低余额止盈**：余额 < 30万 + 收益率 > 1.0% + 其他条件
4. **指数基金止盈**：余额 < 30万 + 收益率 > 3.0% + 其他条件

#### 辅助条件：
- 趋势向上：净值 > 5日均值
- 投资次数较少：times < 5.0
- 估值增长：estimated_change > 0.5%
- 排名合理：rank_100 不为空

### 11. 📈 算法特点

1. **动态止盈点**：根据波动率自动调整止盈阈值
2. **多重条件**：结合收益率、余额、基金类型等多因素
3. **风险控制**：设置最低止盈点(3.0%)防止过早止盈
4. **灵活性高**：关键参数可通过环境变量配置
5. **日志详细**：完整的决策过程日志记录

### 12. ⚠️ 注意事项

1. **趋势判断**：需要净值估算数据的准确性
2. **波动率计算**：依赖基金波动率数据的准确性
3. **余额获取**：有多种余额获取方式，需要确保数据一致性
4. **费率优化**：优先销售低费率份额以降低成本
5. **市场环境**：在牛市中可能需要调整止盈阈值

### 13. 🔄 执行流程总结

1. 检查基础条件（可赎回性、份额存在性）
2. 计算收益率和关键指标
3. 趋势门槛检查（净值 > 5日均值）
4. 动态计算止盈点
5. 检查基本止盈条件
6. 检查低余额相关止盈条件
7. 执行相应的止盈操作
8. 返回操作结果

该算法通过多重条件确保在合适的时机执行止盈操作，既保护收益又考虑资金利用率。
